# Prompt Enhancement Module

## Overview

The `enhance_prompts.py` module adds story context to video generation prompts, creating visual continuity between scenes. It transforms disconnected scene prompts into a cohesive narrative flow.

## What It Does

### Before Enhancement
```
Scene 1: "A desert landscape. The Protagonist stands alone."
Scene 2: "Inside a palace courtyard. The Protagonist sweeps the floor."
```

### After Enhancement
```
Scene 1: "A desert landscape. The Protagonist (thin black stick figure, 
         long dark hair, detached olive face, white dress) stands alone."

Scene 2: "Continuing from: Previous scene: desert, Protagonist, standing alone.
         Scene shifts to palace. Inside a palace courtyard. The Protagonist 
         (thin black stick figure, long dark hair, detached olive face, 
         white dress) sweeps the floor."
```

## Features

✅ **Previous Scene Context** - Each scene references what happened before  
✅ **Character Consistency** - Character descriptions stay the same across all scenes  
✅ **Location Transitions** - Clear hints when location changes or stays the same  
✅ **Emotional Tone** - Detects tone shifts (tense, calm, dramatic, action)  
✅ **Length Control** - Ensures prompts stay under 950 characters (API limit)  
✅ **Simple Integration** - Works as a post-processing step on existing prompts

## Usage

### Command Line

```bash
# Enhance prompts for a video project
python3 utils/enhance_prompts.py \
    outputs/MyVideo/prompts.yaml \
    outputs/MyVideo/spanish_script.txt

# Specify custom output file
python3 utils/enhance_prompts.py \
    outputs/MyVideo/prompts.yaml \
    outputs/MyVideo/spanish_script.txt \
    outputs/MyVideo/prompts_enhanced.yaml
```

### Python API

```python
from utils.enhance_prompts import enhance_prompts

# Enhance prompts (overwrites original file)
success = enhance_prompts(
    prompts_path="outputs/MyVideo/prompts.yaml",
    script_path="outputs/MyVideo/spanish_script.txt"
)

# Or save to a different file
success = enhance_prompts(
    prompts_path="outputs/MyVideo/prompts.yaml",
    script_path="outputs/MyVideo/spanish_script.txt",
    output_path="outputs/MyVideo/prompts_enhanced.yaml"
)
```

## How It Works

### 1. Read Existing Prompts
Reads `prompts.yaml` generated by `multimodal_orchestrator.py`:
- Extracts character profiles with appearance anchors
- Gets all scene prompts (image and video)

### 2. Track Story Context
Uses `story_context.py` to analyze the script:
- Splits script into scene segments
- Extracts location, characters, and action for each scene
- Detects emotional tone using keyword matching

### 3. Enhance Each Prompt
For each scene:
- Adds previous scene context: "Continuing from: [location], [characters], [action]"
- Injects character appearance anchors for consistency
- Adds transition hints: "Scene shifts to [location]" or "Still in [location]"
- Includes emotional tone cues when tone changes significantly

### 4. Ensure Length Limits
- Truncates prompts intelligently if they exceed 950 characters
- Prioritizes: style anchor → character descriptions → scene description → context

## Input Requirements

### prompts.yaml Structure
```yaml
visual_style: "..."
style_subcategory: "..."
character_profile:
  characters:
    - name: "Protagonist"
      appearance_anchor: "thin black stick figure, long dark hair, ..."
    - name: "Pharaoh"
      appearance_anchor: "thin black stick figure, shaved head, ..."
scenes:
  - scene_number: 1
    image_prompt: "..."
    video_prompt: "..."
  - scene_number: 2
    image_prompt: "..."
    video_prompt: "..."
```

### Script File
Plain text file with the story narration:
- Can be English or Spanish (or any language)
- Should have paragraph breaks between major story beats
- Used to extract context for each scene

## Configuration

No configuration needed! The module uses sensible defaults:
- Max prompt length: 950 characters
- Context format: "Continuing from: [location], [characters], [action]"
- Transition detection: automatic based on location and tone changes

## Integration with Pipeline

### Option 1: Manual Enhancement
```bash
# 1. Generate base prompts
python3 utils/multimodal_orchestrator.py ...

# 2. Enhance prompts with story context
python3 utils/enhance_prompts.py \
    outputs/MyVideo/prompts.yaml \
    outputs/MyVideo/spanish_script.txt

# 3. Generate videos

```

### Option 2: Automated in Pipeline
Add to `multimodal_orchestrator.py` after prompt generation:

```python

# After generating prompts...
enhance_prompts(
    prompts_path=prompts_out,
    script_path=script_path
)
```

## Troubleshooting

### "Could not read prompts file"
- Check that `prompts.yaml` exists and is valid YAML
- Make sure you've run `multimodal_orchestrator.py` first

### "Could not extract story context from script"
- Check that script file exists and has content
- The module will continue with empty contexts (no enhancement)

### Prompts too long (>950 chars)
- The module automatically truncates prompts
- If truncation is too aggressive, the script may be too detailed
- Consider simplifying the script or reducing context

### Character descriptions not appearing
- Check that `character_profile` exists in `prompts.yaml`
- Verify each character has an `appearance_anchor` field
- Character names must match between profile and script

## Performance

- **Speed**: Processes 30 scenes in ~1 second
- **Memory**: < 50KB for typical video (30 scenes, 5 characters)
- **Overhead**: Negligible impact on pipeline

## Limitations

- Simple keyword-based emotion detection (no ML/NLP)
- Linear scene progression only (no flashbacks or parallel storylines)
- Character consistency based on appearance_anchor only
- No spatial continuity validation (character positions)

These are intentional simplifications for the MVP. Future versions may add more sophisticated analysis.
